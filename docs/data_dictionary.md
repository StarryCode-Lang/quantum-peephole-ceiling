## Schema Families

The release uses five schema families. The `schema_version` column on each
row identifies which family it belongs to.

| Schema family | Experiments | Notes |
|---------------|-------------|-------|
| `legacy_v2_v3` | E1–E5, E10 | Greedy-bug-fixed + extended. Uses the legacy column names (`original_size`, `optimized_size`). |
| `results_v1` | E11–E13 | Real circuits, compiler baselines, structural ceilings. Introduces `circuit_id`, `circuit_type`, `*_sha256`, and `*_gate_count` aliases. |
| `results_v2` | E14–E18 | Extended suites, multi-compiler, window scaling, connectivity/topology, Clifford+T. Adds extended metrics and per-gate-class reduction columns. |
| `results_v6` | E19–E22, E25 | WCL listing, multi-compiler full, ceiling-aware, industry benchmarks. |
| `results_v7` | E23–E24 | Theory-validation experiments (AG canonical form, Theorem 7 hardness family). Minimal schema with core columns. |

> **Column-name aliasing**: `results_v1`/`results_v2` rows use
> `gate_count_total` / `baseline_gate_count` / `optimized_gate_count` for
> the same concepts that `legacy_v2_v3` rows call `original_size` /
> `optimized_size`. When joining across schemas, map:
> `original_size` ↔ `baseline_gate_count` (≡ `gate_count_total`),
> `optimized_size` ↔ `optimized_gate_count`.

## Common Columns

The following columns appear in **nearly all experiment output CSVs**.
Unless otherwise noted, a column is present in all three schema families.

| Column | Type | Description | Example |
|--------|------|-------------|---------|
| `schema_version` | `str` | Schema family for the row: `legacy_v2_v3`, `results_v1`, `results_v2`. | `results_v2` |
| `experiment_id` | `str` | Unique identifier for the experiment run (e.g., `e10_p1vsp2_001`). | `e10_run_042` |
| `run_id` | `int`/`str` | Monotonic run index within the experiment (`int` in legacy; stable `str` identifier in results_v1/v2). | `42` |
| `circuit_id` | `str` | Stable circuit instance identifier (results_v1/v2 only). | `qft_n5_s01` |
| `circuit_family` | `str` | High-level circuit family (see taxonomy below). | `universal`, `qft`, `ghv` |
| `circuit_type` | `str` | Fine-grained circuit type (results_v1/v2 only). | `qft_full`, `bv_oracle` |
| `n_qubits` | `int` | Number of qubits in the circuit (system size). | `5`, `10`, `20` |
| `depth` | `int` | Original circuit depth (number of layers). | `20`, `50` |
| `seed` | `int` | Random seed used for circuit generation. | `42` |
| `trial` | `int` | Trial index (for repeated runs with same config). | `0`, `1`, `...` |
| `optimizer` | `str` | Optimizer name (see taxonomy below). | `greedy`, `sa`, `hybrid` |
| `original_size` | `int` | Gate count of the original (unoptimized) circuit. Legacy alias of `baseline_gate_count`. | `120` |
| `baseline_gate_count` | `int` | Gate count before optimizer/compiler under comparison (results_v1/v2). Equivalent to `original_size`. | `120` |
| `gate_count_total` | `int` | Input total gate count (results_v1/v2). Same value as `baseline_gate_count` / `original_size`. | `120` |
| `optimized_size` | `int` | Gate count after optimization. Legacy alias of `optimized_gate_count`. | `95` |
| `optimized_gate_count` | `int` | Gate count after optimizer/compiler (results_v1/v2). Equivalent to `optimized_size`. | `95` |
| `reduction` | `float` | Fractional gate reduction: `1 - optimized / baseline`. Range: `[0, 1]`. Proportion unless column ends in `_pct`. | `0.2083` |
| `reduction_pct` | `float` | Percent reduction: `100 * reduction` (results_v1/v2). | `20.83` |
| `fidelity` | `float` | Average gate fidelity between original and optimized circuits. Range: `[0, 1]`. | `0.9992` |
| `success` | `bool` | Binary success flag: `reduction >= 0.20 AND fidelity >= 0.99`. | `True` |
| `runtime_seconds` | `float` | Wall-clock runtime of the optimization (seconds). | `0.342` |
| `timestamp_utc` | `str` (ISO 8601) | UTC timestamp when the row was generated. | `2026-05-15T08:23:17Z` |

## Experiment-Specific Columns

### E1–E5 (legacy_v2_v3)
Covered by [Common Columns](#common-columns) only. No extra columns.
(E1–E5 are the Greedy-bug-fixed baseline experiments.)

### E6–E9 (legacy_v2_v3)
Covered by [Common Columns](#common-columns) only. No extra columns.

### E10: Phase 1 vs Phase 2 Comparison

| Column | Type | Description |
|--------|------|-------------|
| `phase` | `int` | Optimization phase: `1` or `2`. |
| `commutation_depth` | `int` | Number of commutation passes (Phase 2 only). |
| `window_size` | `int` | Commutation window size (Phase 2 only, default 10). Also used by E14/E16. |

### E11: Real Circuit Benchmark

| Column | Type | Description |
|--------|------|-------------|
| `algorithm` | `str` | Algorithm name: `qft`, `ghz`, `bv`, `qaoa`, `vqe`, etc. |
| `source` | `str` | Source library: `qiskit`, `cirq`, `tket`, `custom`. |

### E12: Compiler Baseline

| Column | Type | Description |
|--------|------|-------------|
| `transpiler_level` | `int` | Qiskit transpiler optimization level: `0`, `1`, `2`, or `3`. |
| `compiler` | `str` | Compiler backend: `qiskit`, `cirq`, `tket`. (See also [Multi-Compiler Columns](#multi-compiler-columns) for E15.) |

### E13: Structural Ceiling

See also [Structural Ceiling Columns](#structural-ceiling-columns) below.

| Column | Type | Description |
|--------|------|-------------|
| `theoretical_max` | `float` | Theoretical maximum possible reduction for this circuit structure. |
| `ceiling_gap` | `float` | Gap between achieved and theoretical maximum reduction. |

### E14–E18: Extended Benchmarks

| Column | Type | Description |
|--------|------|-------------|
| `benchmark_suite` | `str` | Suite version: `v5_extended`, `v5_clifford_t`. |
| `connectivity` | `str` | Hardware connectivity: `all_to_all`, `linear`, `heavy_hex` (E17 only). (E17 also reports `topology` — see [Topology Columns](#topology-columns).) |
| `gate_set` | `str` | Gate set used: `universal`, `clifford_t` (E18 only). |

## Circuit Family Taxonomy

### Synthetic (Random) Families

| Family | Description | Parameters |
|--------|-------------|------------|
| `universal` | Random universal gate set circuit. | `n_qubits`, `depth`, `seed` |
| `structured` | Structured brickwork (layered) circuit. | `n_qubits`, `depth`, `seed` |
| `random_clifford` | Random Clifford circuit. | `n_qubits`, `depth` |
| `haar_random` | Haar-random unitary circuit. | `n_qubits`, `depth` |
| `iqp` | Instantaneous Quantum Polynomial-time circuit. | `n_qubits`, `depth` |

### Algorithmic Families (Real Benchmarks)

| Family | Description | Parameters |
|--------|-------------|------------|
| `qft` | Quantum Fourier Transform. | `n_qubits` |
| `ghz` | Greenberger-Horne-Zeilinger state preparation. | `n_qubits` |
| `bv` | Bernstein-Vazirani oracle. | `n_qubits` |
| `qaoa` | Quantum Approximate Optimization Algorithm ansatz. | `n_qubits`, `layers` |
| `vqe` | Variational Quantum Eigensolver ansatz. | `n_qubits`, `layers` |
| `grover` | Grover's search algorithm. | `n_qubits`, `iterations` |
| `quantum_adder` | Quantum ripple-carry adder. | `n_qubits` |
| `quantum_walk` | Quantum random walk. | `n_qubits`, `steps` |
| `surface_code` | Surface code syndrome extraction. | `n_qubits`, `rounds` |
| `uccsd` | Unitary Coupled Cluster with Singles and Doubles. | `n_qubits`, `depth` |

## Optimizer Taxonomy

| Optimizer | Abbreviation | Phase | Description |
|-----------|--------------|-------|-------------|
| Greedy Gate Cancellation | `greedy` | 1 | Deterministic adjacent self-inverse pair removal. |
| Simulated Annealing | `sa` | 1 | Metropolis-Hastings with temperature annealing. |
| Genetic Algorithm | `ga` | 1 | Tournament selection + patch-based crossover. |
| Random Local Search | `rls` | 1 | Random neighbor generation with early stopping. |
| Commutation Rewriter | `commutation` | 2 | Swap-based commutation to bring inverse gates adjacent. |
| Hybrid (Greedy→Commutation→Greedy) | `hybrid` | 1+2 | Three-stage pipeline. |

## Data Types and Constraints

### Integer Columns
- `run_id`: `>= 0`
- `n_qubits`: `>= 1` (physically meaningful: `>= 2`)
- `depth`: `>= 1`
- `seed`: `>= 0`
- `trial`: `>= 0`
- `original_size` / `baseline_gate_count` / `gate_count_total`: `>= 1`
- `optimized_size` / `optimized_gate_count`: `>= 0` (can be 0 if all gates cancel)
- `optimized_depth`: `>= 0`
- `optimized_2q_gates`: `>= 0`
- `optimized_cnot_count`: `>= 0`
- `baseline_t_count`: `>= 0`
- `optimized_t_count`: `>= 0`
- `n_edges`: `>= 0`

### Float Columns
- `reduction`: `0.0 <= reduction <= 1.0`
- `reduction_pct`: `0.0 <= reduction_pct <= 100.0`
- `fidelity`: `0.0 <= fidelity <= 1.0`
- `runtime_seconds`: `>= 0.0`
- `depth_reduction`: `0.0 <= depth_reduction <= 1.0`
- `two_qubit_reduction`: `0.0 <= two_qubit_reduction <= 1.0`
- `cnot_reduction`: `0.0 <= cnot_reduction <= 1.0`
- `t_count_reduction`: `0.0 <= t_count_reduction <= 1.0`
- `theoretical_max`: `0.0 <= theoretical_max <= 1.0`
- `ceiling_gap`: `>= 0.0`
- `structural_upper_bound_reduction`: `0.0 <= value <= 1.0`
- `observed_best_reduction`: `0.0 <= value <= 1.0`

### Boolean Columns
- `success`: Computed as `reduction >= 0.20 AND fidelity >= 0.99`

## Missing Value Policy

| Scenario | Handling |
|----------|----------|
| Failed optimization | `optimized_size = original_size`, `reduction = 0.0`, `fidelity = NaN` |
| Timeout | `optimized_size = original_size`, `reduction = 0.0`, `fidelity = NaN`, `runtime_seconds = timeout_value` |
| Fidelity > 1.0 (numerical noise) | Clamped to `1.0` |
| Fidelity < 0.0 (numerical noise) | Clamped to `0.0` |

## Provenance and Versioning

All data files include provenance columns (appended by `src/provenance.py`):
`git_commit`, `source_hash`, `input_circuit_sha256`, `output_circuit_sha256`,
`python_version`, `qiskit_version`, `numpy_version`, etc.

Data versions:
- **v2_fixed**: E1–E5 (Greedy bug fixed)
- **v3_extended** (superseded → v5): E10
- **v4**: E11–E13 (real circuits, compiler baselines, structural ceilings)
- **v5**: E10 (expanded), E14–E18 (extended suites, multi-compiler, window scaling, connectivity, Clifford+T)
